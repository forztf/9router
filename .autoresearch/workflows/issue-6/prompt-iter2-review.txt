## Issue 需求

## Description
新增 GET /api/usage/apikey-stats 接口，从 usageDaily 的 byApiKey 聚合数据中提取按 API Key 维度的统计，关联 apiKeys 表获取 name。

不修改现有表结构、schema 或 migration。

## Acceptance Criteria
- 新增 GET /api/usage/apikey-stats 接口，接受 period 参数
- 返回格式包含：每个 apiKey 的 name、requests、promptTokens、completionTokens、cost
- 返回格式包含：每个 apiKey 的 byProvider 和 byModel 拆分
- 返回汇总数据：totalRequests、totalPromptTokens、totalCompletionTokens、totalCost
- 无数据时返回空数组而非 404
- period 参数校验：只接受 24h/7d/30d/60d
- 不修改现有表结构、schema、migration
- Typecheck/lint passes

## 实现改动摘要

修改了3个文件，新增1个文件，共 110 行新增代码：

1. **usageRepo.js**: 新增 `maskApiKey()` 脱敏函数和 `getApiKeyStats(period)` 异步函数
   - maskApiKey: local-no-key → "Local (No API Key)", 短key → "sk-***", 长key → "sk-***xxxx" 格式
   - getApiKeyStats:
     - 24h 模式: 直接查 usageHistory 表，按 apiKey 字段分组
     - 7d/30d/60d 模式: 用 loadDaysInRange 从 usageDaily 表获取日数据，遍历 data.byApiKey
     - byApiKey key 格式为 `apiKeyValue|model|provider`，取 ak.apiKey 或用 | split 取第一段
     - 按 apiKey 分组汇总 requests/promptTokens/completionTokens/cost
     - 每个 apiKey 下保留 byProvider 和 byModel 二级拆分
     - 调用 apiKeysRepo.getApiKeys() 获取 name 映射
     - 计算汇总 totalRequests/totalPromptTokens/totalCompletionTokens/totalCost
     - 返回 items 数组（按 cost 降序）+ 汇总字段

2. **db/index.js**: 导出 getApiKeyStats
3. **usageDb.js**: 导出 getApiKeyStats

4. **src/app/api/usage/apikey-stats/route.js**: 新增 GET handler
   - period 参数默认 7d
   - 校验只接受 24h/7d/30d/60d，非法返回 400
   - 调用 getApiKeyStats(period)
   - 错误返回 500
   - export const dynamic = 'force-dynamic'
