Issue 标题: [Bug] API Key 统计：同名 key 被分拆为多条 + 显示 sk-* 而非名称
Issue 内容:
## 问题描述

API Key 统计页面显示混乱：同一 API Key 出现多条记录，原因是 key 有三种格式（13/35/70 char）导致被当作不同 key 处理。

**当前症状：**
- 同一 API Key 显示为 3 条记录（分别对应 35-char / 70-char / 13-char）
- 显示的是 `sk-***` 而非 API Key 名称（用户创建的命名）

**需要修复：**
1. **合并同名 API Key**：按前 8 字符或名称归一化，同一 key 的请求数合并到一条
2. **显示 Key 名称**：统计展示用 Key 创建时的名称，而非 `sk-*` 掩码
3. **追踪来源**：分析 13/35/70 char 三种格式的生成位置，确认是否 proxy 层/上游转发引入的差异

---

## 技术背景

当前统计 endpoint: `GET /api/usage/apikey-stats`

数据库表 `api_keys` 有字段：`id, name, key_prefix, key_hash, created_at`

但 `/api/usage/apikey-stats` 的返回格式缺少 `name` 字段，只返回了 `maskedKey`。
